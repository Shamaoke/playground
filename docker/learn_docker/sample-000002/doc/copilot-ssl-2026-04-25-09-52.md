### 💬 Copilot

The `-subj` option specifies the **Subject Distinguished Name (DN)** for the certificate. Here's what you need to know:

## Format
```
-subj "/C=CC/ST=STATE/L=CITY/O=ORGANIZATION/CN=COMMONNAME"
```

## Field Meanings

| Code | Field | Example | Purpose |
|------|-------|---------|---------|
| **C** | Country | US | 2-letter ISO country code |
| **ST** | State/Province | California | Full state or province name |
| **L** | Locality/City | San Francisco | City name |
| **O** | Organization | MyCompany | Company/organization name |
| **CN** | Common Name | example.com | Domain name (critical for HTTPS) |

## Optional Fields
- **OU** — Organizational Unit (department, e.g., "IT Department")
- **emailAddress** — Email address (rarely used in modern certs)

## Example for a Self-Signed Cert
```bash
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 \
  -subj "/C=US/ST=California/L=San Francisco/O=MyCompany/CN=localhost"
```

## Key Points
- **CN is critical** — For local development, use `localhost` or `127.0.0.1`; for production, use your actual domain
- **Order matters** — Always follow C→ST→L→O→CN order for compatibility
- **No spaces** around slashes — `/C=US` not `/ C = US /`
- **First slash is required** — Start with `/C=...`
- **Escape special characters** — Use backslash for characters like commas or quotes

For a local development certificate, a minimal example is:
```bash
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 \
  -subj "/C=US/ST=State/L=City/O=Organization/CN=localhost"
```
